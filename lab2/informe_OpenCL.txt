Informe:
  El presente informe describe la implementación y decisiones de diseño tomadas 
respecto de paralelizar procedimientos en OpenCL.

Estructura del Código:
  El código se divide en 4 archivos por en los ejercicios 2, 3 y 4, dejando el 
ejercicio 1 en 3 archivos pues es e único que no lee código kernel.Ccada uno 
dentro de su carpeta con el nombre de ejercicio.
  Los archivos son:
    - ejercicio*.cpp: Contiene la función main de cada ejercicio.
    - helpers.h: Contiene las definiciones de cada funcion auxiliar que se uso 
para modularizar.
    - helpers.cpp: Contiene la implementación de cada función definida en 
helpers.h.
    - kernel.cl: Contiene el código Kernel que se va a utilizar en cada caso 
para ser leidos por la función readKernel().

Diseño:
  Se decidio modularizar las funciones de OpenCL API dentro del archivo helpers, 
para así poder distinguir con mayor claridad el flujo de trabajo dentro del 
main, dejando así, solo variables declaradas y funciones que llaman a las 
implementadas en el archivo helpers.
  Se utilizo tambien la función readKernel() para poder leer el código Kernel 
desde el archivo kernel.cl evitando así declarar en una variable todo el código 
de Kernel y dejar solo el código con funciones.
  Para obtener un mejor control del código y a modo de testear la 
implementacíon, se utlizo la tecnica de programación defensiva con control de 
errores en todas aquellas funciones que devuelvan un código de error, 
utilizando printf y un mensaje de error en caso de error, o en caso contrario 
un mensaje de éxito.
